## 1. 概要
MidnightブロックチェーンのZKPを活用した、3ラウンド制の戦略的カードバトル。

## 2. ゲームモード
- **シングルプレイ (Solo):** CPUを相手に対戦。ルール習得や練習に最適。
- **マルチプレイ (Multiplayer):** WebSocketを利用したリアルタイム対戦（未実装）。

### オンチェーンモード
各ゲームモードに「On-Chain Mode」チェックボックスを追加。

| 設定 | 動作 |
|---|---|
| Off（デフォルト） | オフチェーンで完結。TX なし。 |
| On | 勝敗ロジックをスマートコントラクトで実行。TX ハッシュ・コントラクトアドレスを表示。 |

- CPU 対戦・オンライン対戦の**両方**で利用可能。
- **現在はダミー実装**（フェイク TX ハッシュ・コントラクトアドレスを生成）。
- Phase 3 で実際の Midnight コントラクト呼び出しに置き換え予定。

## 3. 通信仕様 (Multiplayer — 未実装)
- **Real-time Sync:** WebSocketを通じて、カードの提出、秘匿選択、バトル/フォールドの決断を同期。
- **Commitment Flow:** 
  1. 片方のプレイヤーがカードを提出。
  2. サーバーが「相手が提出済み」であることをもう一方に通知。
  3. 両者が提出完了後、必要に応じて意思決定フェーズ（Battle/Fold）に移行。（Battle/Fold）の選択は相手が「秘匿」を出した時だけ。自分が「秘匿」を出した場合は選択しない。

## 2. 勝利点 (Victory Points) 設計
各ラウンドの対戦モードと結果に応じ、以下の通りVPを付与する。

| 状況 | 勝者 (A) | 敗者 (B) | 設計意図 |
| :--- | :---: | :---: | :--- |
| **公開 vs 公開** | **+1 VP** | 0 | 標準的な対戦。 |
| **秘匿(A) vs 公開(B) (Battle)** | **+1 VP** | 0 | Aは秘匿の恩恵（手札バレ防止）を得るが、加点は標準的。 |
| **公開(A) vs 秘匿(B) (Battle)** | **+1 VP** | **-1 VP** | Bは秘匿のリスク（敗北ペナルティ）を負う。 |
| **秘匿 vs 秘匿 (Both Battle)** | **+3 VP** | **-1 VP** | 互いにリスクを負った決着。勝者は高得点。 |
| **秘匿(A) vs 公開(B) (Fold)** | **0 VP** | **0 VP** | Bの防御。 |
| **引き分け (Draw)** | **+1 VP** | **+1 VP** | 同じ手札の場合。 |

- **ペナルティ:** 「秘匿」を選択して敗北した場合、常に **-1 VP** となる。
- **フォールドの仕様:** 相手が「秘匿」の時、自分が降りれば両者 0 VP。これにより、強いカードを持っている側の加点機会を奪うことができる。

## 3. ゲームルール詳細
### 3.1 手札
- 4種（グー・チョキ・パー・負け犬）からランダムに3枚（「負け犬」は必ず1枚）。
- 各カードは使い切り。

### 3.2 対戦モードの選択
1. **公開 (Public):** 手札をオープンにして戦う。
2. **秘匿 (Hidden):** 手札を伏せ、ZKPで勝敗を証明する。

### 3.3 フォールドのルール
相手が「秘匿」を選択した場合、もう一方は「対戦 (Battle)」か「フォールド (Fold)」を選択できる。
- **フォールドのメリット:** 相手の「秘匿勝利 (+1 or +3 VP)」を阻止。
- **フォールドのデメリット:** 自分が勝っていた可能性があっても 0 VP となる。ただし自分の「敗北ペナルティ (-1 VP)」を回避できる。

## 4. 戦略的インセンティブ
- **「秘匿」のジレンマ:** 
  - 高得点を狙いたいが、相手に「勝てない」と判断されてフォールドされると、本来「公開」なら得られたはずの +1 VP すら得られなくなる。
  - そのため、あえて「公開」して確実に 1点を取りに行くか、欲張って「秘匿」にするかの選択が生まれる。
- **ブラフとしての「秘匿」:** 
  - 自分が「負け犬」の時に「秘匿」で出し、相手を怖がらせてフォールドさせれば、実質的な負けを 0点（無効試合）に持ち込むことができる。

## 5. 対戦フロー
1. **Onboarding (Connect Wallet):**
   - 「Connect Wallet (Demo)」ボタンでダミーウォレットを接続（アカウント作成・Mnemonic 生成機能は廃止）。
   - Lace Wallet が検出されている場合は「Connect Lace Wallet」ボタンも表示。
2. **Mode Select:** ゲームモード（Solo / Multi）とオンチェーンモードのON/OFFを選択。
3. Round 1~3:
   - **Step 1 (Card Selection):** 各プレイヤーが手札からカードを選び、「公開」または「秘匿」を指定して提出。
   - **Step 2 (Decision Phase):** 相手が「秘匿」を選択した場合、プレイヤーは「対戦 (Battle)」か「フォールド (Fold)」を選択。
   - **Step 3 (Resolution):** 両者のカードが開示（またはZKP検証）され、勝敗とVPが確定。「秘匿」のカードは最後まで公開しない。
   - **Next:** 手札が残っている場合は次のラウンドへ。Multiの場合は相手の通信を待つ。

4. **Final Result:** 3ラウンド終了後、合計VPが多いプレイヤーが勝者。勝敗が決まった時点で通信を切断する。
5. **Reward:** 勝者に **10 YTTM トークン（Shielded）** を付与。
   - YTTM は ZKP シールドトークン。第三者には残高・送金先が非公開。
   - 受取者本人のみ残高を確認できる。

## 6. トークン仕様 (YTTM)
- **種別:** Midnight Shielded Token（ZKP による秘匿）
- **取得方法:** ゲーム勝利時のみ mint（スマートコントラクトが発行）
- **可視性:** 第三者には残高・送金額が見えない。受取者のみ確認可能。
- **現在の実装:** ダミー（フェイク残高加算）。Phase 3 で yttm.compact に置き換え予定。
